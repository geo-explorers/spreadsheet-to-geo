```typescript
import XLSX from 'xlsx';
// Uses XLSX.readFile(), XLSX.utils.sheet_to_json(sheet, { defval: '' })
// Returns { ids, spaceId, errors } -- accumulates all errors

function getColumnValue(row: Record<string, unknown>, headerName: string): unknown {
  if (headerName in row) return row[headerName];
  for (const key of Object.keys(row)) {
    const cleaned = key.replace(/^\uFEFF/, '');
    if (cleaned === headerName) return row[key];
  }
  return undefined;
}
```

  <action>
Create `src/config/merge-types.ts` with these type definitions:

1. **MergeOptions** -- CLI options for the merge command:
   - `network?: string` -- optional network flag
   - `dryRun: boolean`
   - `output: string` -- report output directory
   - `verbose: boolean`
   - `yes: boolean` -- skip confirmation
   Follows the same pattern as `DeleteOptions` and `UpdateCommandOptions`.

2. **MergePair** -- single keeper/merger pair from the Excel template:
   - `keeperName: string` -- keeper entity name from spreadsheet
   - `mergerName: string` -- merger entity name from spreadsheet
   - `rowNumber: number` -- 1-based row number for error reporting

3. **MergeConflict** -- when keeper and merger both have a property with different values:
   - `propertyId: string`
   - `propertyName: string`
   - `keeperValue: string` -- human-readable keeper value
   - `mergerValue: string` -- human-readable merger value

4. **MergePairDiff** -- computed diff for a single merge pair (central type for the diff engine):
   - `keeperName: string`
   - `keeperId: string`
   - `mergerName: string`
   - `mergerId: string`
   - `propertiesToTransfer: Array<{ propertyId: string; propertyName: string; mergerValue: string; typedValue: TypedValue }>` -- unique properties to copy from merger
   - `conflicts: MergeConflict[]` -- properties both have with different values
   - `relationsToRepoint: Array<{ relationId: string; direction: 'outgoing' | 'incoming'; typeId: string; otherEntityId: string; otherEntityName: string }>` -- relations to delete+recreate
   - `relationsSkipped: Array<{ direction: 'outgoing' | 'incoming'; typeId: string; otherEntityId: string; otherEntityName: string }>` -- keeper already has equivalent
   - `typesToTransfer: string[]` -- type IDs merger has that keeper doesn't
   - `mergerDeleteOps: Op[]` -- from buildDeleteOps() for post-transfer cleanup
   Import `TypedValue` from `@geoprotocol/geo-sdk` and `Op` from `@geoprotocol/geo-sdk`.

5. **MergeSummary** -- aggregate counts across all pairs:
   - `totalPairs: number`
   - `propertiesTransferred: number`
   - `conflictsDetected: number`
   - `relationsRepointed: number`
   - `relationsSkipped: number`
   - `typesTransferred: number`
   - `mergersDeleted: number`

6. **MergeBatch** -- ops for a single pair ready for publishing:
   - `ops: Op[]`
   - `pairDiff: MergePairDiff` -- the diff that produced these ops

<task type="auto">
  <name>Task 2: Build Merge tab Excel parser in merge-parser.ts</name>
  <files>src/parsers/merge-parser.ts</files>
  <action>
Create `src/parsers/merge-parser.ts` that parses the Excel merge template.

**Template format** (per CONTEXT.md locked decisions):
- **Metadata tab**: Contains `Space ID` (field/value rows), `Operation type: MERGE`, same format as upsert/update templates
- **Merge tab**: Two columns with headers `Keeper` and `Merger`, each row is a keeper/merger entity name pair

**Function: `parseMergeTemplate(filePath: string)`**

Returns `{ pairs: MergePair[]; spaceId: string; operationType: string; errors: string[] }`

Implementation:
1. Read workbook via `XLSX.readFile(filePath)`
2. Parse Metadata tab:
   - Use `XLSX.utils.sheet_to_json(sheet, { defval: '' })` to get rows
   - Metadata tab uses Field/Value row pattern (same as excel-parser.ts's parseMetadataTab)
   - Extract `spaceId` from the row where Field = "Space ID"
   - Extract `operationType` from the row where Field = "Operation type"
   - If Metadata tab is missing, add error: `'Tab "Metadata" not found in workbook'`
   - If Space ID is missing/empty, add error: `'Space ID not found in Metadata tab'`
3. Parse Merge tab:
   - If tab is missing, add error: `'Tab "Merge" not found in workbook'`
   - Read rows with `XLSX.utils.sheet_to_json(sheet, { defval: '' })`
   - Use a `getColumnValue()` helper (same BOM-tolerant pattern as entity-id-parser.ts) to read `Keeper` and `Merger` columns
   - For each row:
     - Trim whitespace from both values
     - Skip completely blank rows (both columns empty)
     - If keeper is present but merger is empty, add error: `Row N: Missing Merger entity name`
     - If merger is present but keeper is empty, add error: `Row N: Missing Keeper entity name`
     - If both present and non-empty, create a `MergePair` with `keeperName`, `mergerName`, and `rowNumber` (1-based: row index + 2 for header)
   - Validate: keeper and merger cannot be the same name (case-insensitive). Error: `Row N: Keeper and Merger cannot be the same entity ("name")`
4. Return accumulated results -- follow the error-accumulation pattern from entity-id-parser.ts (collect all errors, don't fail on first)

Import `MergePair` from `../config/merge-types.js`. Import `cleanString` from `../utils/cell-parsers.js` for whitespace trimming.

Use the same `getColumnValue()` BOM-tolerant helper as entity-id-parser.ts (copy the function locally into this file, same as entity-id-parser does -- don't extract to shared module, keeping parsers self-contained).

The Metadata tab Field/Value parsing pattern: the sheet has two columns (`Field` and `Value`). Each row is a key-value pair. Iterate rows and match `Field` values (case-insensitive) against known field names to extract `Space ID`, `Operation type`, etc. Use `cleanString()` on extracted values.
